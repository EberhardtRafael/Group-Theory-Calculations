# Internationalization (i18n)

This project uses `next-intl` for internationalization support.

## Current Status
- **Supported locales**: English (en) only
- **Default locale**: en
- **Locale detection**: Automatic via middleware

## Adding New Languages

1. **Create a new locale file** in `/messages/`:
   ```bash
   # Example for Spanish
   cp messages/en.json messages/es.json
   ```

2. **Translate the content** in the new file

3. **Update the locale configuration** in:
   - `/src/config/locales.ts`: Add to `SUPPORTED_LOCALES` array
   - `/src/middleware.ts`: Add to `locales` array

4. **That's it!** The app will automatically detect and support the new locale.

## Usage in Components

```tsx
import { useTranslations } from 'next-intl';

const MyComponent = () => {
  const t = useTranslations('namespace');
  
  return <div>{t('key')}</div>;
};
```

## File Structure

```
messages/
  └── en.json          # English translations
src/
  ├── i18n.ts          # i18n configuration
  ├── middleware.ts     # Locale detection middleware
  └── config/
      └── locales.ts   # Supported locales configuration
```
